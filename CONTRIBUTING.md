# Contributing

Thanks for considering contributing to TinyPdf!

- Fork the repository and create a feature branch.
- Run tests with `dotnet test`.
- Follow the existing coding style and keep changes minimal and focused.
- Add unit tests for bug fixes and new features.
- Ensure benchmarks remain deterministic; run them in `Release` for performance comparisons.
- Open a pull request describing your change.

Maintainers will review and merge or request changes. For security issues, see `SECURITY.md`.
